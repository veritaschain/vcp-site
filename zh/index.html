什么是 VeritasChain Protocol (VCP)?

VeritasChain Protocol (VCP) 是一个开放标准规范，用于以不可变和可验证的格式记录算法交易和AI驱动交易的「决策制定」和「执行结果」。VCP提供加密保护的证据链，确保符合国际法规，包括MiFID II、EU AI Act和CAT (综合审计跟踪)，是RegTech、审计和合规的关键技术。

VCP is an open protocol for recording the "decision-making" and "execution results" of algorithms and AI-driven trading in an immutable and verifiable format, designed to support global regulatory requirements such as MiFID II, EU AI Act, and CAT.

VCP还记录以下内容:

与仅记录订单和执行的FIX不同:

  • AI模型版本和决策因素
  • 有效的风险限制和触发的控制
  • 时间同步 (PTP / NTP) 时间戳
  • 密码学封装的审计轨迹

这扩展了FIX无法记录的内部决策因素和风险状态。
这使VCP对MiFID II RTS 25、EU AI Act Article 12和下一代RegTech合规至关重要。

为什么是现在?

全球监管机构正在将算法和AI驱动交易的监督从"基于信任"转向"基于验证"。

欧盟境内算法交易系统需要不可变的审计轨迹

🇪🇺

EU AI Act (2024/2026)

包括交易算法在内的高风险AI系统需要可解释性和可审计性

🇺🇸

CAT Rule 613 (持续)

美国SEC的综合审计跟踪要求详细的交易生命周期跟踪

🌏

亚太地区协调

日本、新加坡和香港正在与全球审计标准保持一致

VCP提供满足所有这些要求的单一全球标准，防止跨司法管辖区的碎片化 — by design。

技术栈

1. Event ID
UUIDv7 (RFC 9562, 时间顺序, v4回退)
2. Signature
Ed25519 / ECDSA / Dilithium (PQC)
3. Verification
Merkle Tree + Chain Validation
4. Timestamp
PTP (IEEE 1588-2019) / NTP (Chrony)
Platinum: <1µs | Gold: <1ms
Silver: Best-effort (系统时间; 无精度保证)
5. Storage
SBE / JSON / Parquet / FlatBuffers
Zero-Copy / Kernel Bypass / RDMA ready
6. Integration
FIX Protocol Sidecar

VeritasChain Certified (VC-Certified)

VCP Compliance Certification

VeritasChain Inc. (VeritasChain株式会社) 托管VeritasChain Protocol (VCP)标准并运营VeritasChain Certified (VC-Certified)计划，这是全球算法交易和AI驱动交易系统的认证计划，促进AI安全、RegTech和FinTech行业的标准化。

VeritasChain Inc. hosts the VeritasChain Protocol (VCP) standard and runs the VeritasChain Certified (VC-Certified) program.

VC-Certified验证交易系统:

  • 实现完整的VCP事件模型 (SIG → ORD → ACK → EXE → CLS)
  • 在所有事件中保持哈希链和Merkle树完整性
  • 满足其层级的时钟同步要求 (Silver / Gold / Platinum)
  • 记录AI模型版本、决策因素和风险参数

VC-Certified确保交易系统无法伪造、更改或遗漏审计数据。

VC-Certified包括覆盖所有VCP模块的自动化一致性测试(CTS): VCP-CORE、VCP-TRADE、VCP-GOV、VCP-RISK、VCP-PRIVACY和VCP-RECOVERY。
