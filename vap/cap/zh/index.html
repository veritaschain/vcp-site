为什么需要CAP

AI驱动的创意工作流中的结构性挑战

权利来源不透明

输入AI工作流的素材的权利依据无法追溯。当争议发生时，无法证明合法使用。

同意模糊

是否获得了训练或生成的同意不明确。声纹克隆、肖像使用、风格模仿在没有适当授权记录的情况下进行。

机密分类缺失

未发布的角色、故事设定和资产在没有机密分类的情况下输入AI，存在通过训练数据泄露的风险。

CAP如何解决这些挑战

挑战 描述 CAP解决方案
权利不透明 AI输入素材的权利依据无法追溯 RightsBasis字段记录权利基础
同意模糊 训练/生成的同意状态不明确 ConsentBasis加密固定同意状态
机密缺失 未发布素材的敏感性未管理 ConfidentialityLevel分类敏感度
责任不明 争议中无法确定责任方 User/Role记录执行者身份
篡改风险 事后记录可被篡改 Hash Chain提供加密保证

威胁模型

CAP应对的五类威胁

TH-1 IP稀释

独特的表达风格和世界观被AI轻易模仿，品牌价值稀释，市场差异化困难。

对应字段: AssetID, RightsBasis
TH-2 逆向流出

第三方使用用您的素材训练的AI，产生与您产品相似的市场输出。

对应字段: AssetID, ModelContext
TH-3 机密泄露

未发布的角色、设定、素材、代码被输入AI，通过训练数据泄露到外部。

对应字段: ConfidentialityLevel, User
TH-4 深度伪造/人格侵害

私人图片/视频未经同意用于生成：诽谤、色情内容、骚扰。

对应字段: ConsentBasis, AssetID
TH-5 品牌/风格模仿

企业官网/IR/财报设计和风格被模仿，产生被误认为官方的输出。

对应字段: RightsBasis, PermittedUse

INGEST

代码: 1 — 素材输入

记录输入AI工作流的素材、数据和参考信息。

必填字段:

  • • AssetID, RightsBasis, ConfidentialityLevel
  • • User, Timestamp

TRAIN

代码: 2 — 模型训练

记录模型训练、微调和追加训练活动。

必填字段:

  • • ModelContext, TrainingType, InputAssetIDs
  • • User, Timestamp

GEN

代码: 3 — 内容生成

记录AI生成的内容创建。

必填字段:

  • • ModelContext, OutputAssetID, PromptHash
  • • User, Timestamp

EXPORT

代码: 4 — 外部输出

记录生成内容的输出、分发和交付。

必填字段:

  • • AssetID, Destination, PermittedUse
  • • User, Timestamp
