VCC — VeritasChain Cloud

VCP加密日志服务

VCC是基于云的VCP日志记录和证书颁发服务。

提供加密可验证的审计追踪， 符合VCP v1.1标准。

通过交互式演示体验VCP的实际运行。

什么是VCC？

VeritasChain Cloud为VCP合规系统提供基于云的日志记录和证书颁发

核心目的

VCC是VeritasChain Protocol (VCP)的参考实现，演示了：

  • 客户端VCP v1.1事件创建和管理
  • RFC 6962合规默克尔树实现
  • 证书生成和验证
  • 基于IndexedDB的持久存储

目标用户

  • 评估VCP集成的开发者
  • 了解VCP数据结构的审计师
  • 探索透明度解决方案的自营交易公司
  • 研究合规性的监管科技研究人员

VCP v1.1合规性

VCC演示实现了VCP v1.1定义的三层完整性架构

1

事件完整性

单个事件的防篡改证据

  • EventHash (SHA-256)
  • PrevHash (可选)
2

集合完整性

使用默克尔树的批量验证

  • 默克尔树 (RFC 6962)
  • MerkleRoot生成
  • 审计路径生成
3

外部可验证性

第三方时间戳锚定

  • 外部锚定 (模拟)
  • 数字签名 (SHA-256)

支持的事件类型

1 SIG — 信号生成 2 ORD — 订单提交 3 ACK — 订单确认 4 REJ — 订单拒绝 5 EXE — 执行 6 CXL — 取消 10 CLS — 平仓

加密实现

VCC演示使用行业标准加密原语

SHA-256哈希

使用Web Crypto API实现原生性能：

const sha256 = async (data) => {
  const encoder = new TextEncoder();
  const dataBuffer = encoder.encode(
    typeof data === 'string' ? data : JSON.stringify(data)
  );
  const hashBuffer = await crypto.subtle.digest('SHA-256', dataBuffer);
  return Array.from(new Uint8Array(hashBuffer))
    .map(b => b.toString(16).padStart(2, '0'))
    .join('');
};

安全注意事项

了解演示版的局限性和生产环境的替代方案

重要提示

VCC演示是一个演示工具。"锚定"在浏览器中模拟， 不提供外部可验证性。生产系统必须使用真实的外部时间戳机构 （OpenTimestamps、RFC 3161）、用于不变性的区块链锚定和基于HSM的数字签名。

客户端限制

方面 VCC演示 生产环境
密钥存储 无 HSM
签名 SHA-256 Ed25519
防篡改 浏览器 区块链
数据隐私 本地 服务器端

安全功能

  • Web Crypto API

    通过crypto.subtle的原生加密操作

  • 安全随机

    用于UUID生成的crypto.getRandomValues()

  • 同源策略

    浏览器强制执行的跨站数据访问防护

  • 用户控制

    用户可以随时清除所有数据
